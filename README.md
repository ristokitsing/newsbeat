# newsbeat-app

Native SwiftUI reader for the static feed published by the separate
`newsbeat-digest` repository.

The app supports macOS and iOS from one shared codebase. It reads
`feed/digest.json`; it does not collect, rank, summarize, or publish stories,
and it never accesses the processor's SQLite database.

```text
src/
├── shared/    # Feed models, loading, cache, reader UI, post generation
├── macOS/     # macOS actions and optional local processor coordinator
└── iOS/       # iOS copy/share behavior
tests/macOS/
project.yml
```

## Requirements

- Xcode 26.5 or newer
- Swift 6.3.2
- XcodeGen

Generate the project after changing `project.yml`:

```bash
xcodegen generate
```

`Newsbeat.xcodeproj` and `generated/` are reproducible XcodeGen outputs and
are intentionally ignored. Keep source changes in `project.yml` and `src/`.

Build and test:

```bash
xcodebuild -project Newsbeat.xcodeproj \
  -scheme NewsbeatMac \
  -destination 'platform=macOS' \
  build test

xcodebuild -project Newsbeat.xcodeproj \
  -scheme NewsbeatIOS \
  -sdk iphoneos \
  -destination 'generic/platform=iOS' \
  CODE_SIGNING_ALLOWED=NO \
  build
```

Select your development team in Xcode before installing on a physical iPhone
or distributing either target.

## Feed configuration

Open Settings and select either:

- **Local file**: choose a `digest.json` generated by `newsbeat-digest`. This
  is the default on macOS.
- **Remote URL**: enter the HTTPS URL of a hosted static feed. This is the
  default on iOS.

The default macOS local path is:

```text
~/Projects/newsbeat/feed/digest.json
```

That assumes the digest repository remains checked out as `newsbeat` beside
this repository. The path is configurable.

Every successful load is saved atomically under Application Support. If the
next local or remote load fails, the app opens the cached feed and labels it
as offline data.

## Reader and post generation

The app groups stories by `digest_date`. Story details include the summary,
importance, caution, source link, and LinkedIn/Instagram actions. iOS also
provides native share sheets.

LinkedIn and Instagram drafts are generated on demand from the stored story
summary. The app calls the Anthropic Messages API directly, caches generated
posts under `Application Support/newsbeat/posts-cache.json`, and supports
copy and regeneration.

Configure the Anthropic API key and model in **Settings -> Post generation**.
The key is stored in the device Keychain and never embedded in source or the
feed. The model defaults to `claude-haiku-4-5`.

## macOS local host mode

The macOS app can optionally launch the separate processor while the app is
open. Configure:

- digest repository path, normally `~/Projects/newsbeat`
- Python executable, normally `<digest repository>/.venv/bin/python`

The coordinator:

- invokes `python -m newsbeat_digest run` in the configured digest repository
- runs when a local feed is stale and checks the 07:00, 11:00, 16:00, and
  21:00 local schedule
- prevents overlapping runs and reports command output
- reloads `<digest repository>/feed/digest.json` after success
- stops scheduling when the app exits

The macOS target deliberately has App Sandbox disabled because this personal
tool launches a user-configured executable and reads another repository. The
iOS target does not compile or contain the host coordinator.

For VPS-hosted operation, select **Remote URL** and disable local host mode.
No reader or feed-schema change is required.

## Repository boundary

The stable integration boundary is `digest.json` version 1. Feed producer
changes belong in `newsbeat-digest`; decoder and UI changes belong here.
Schema changes must be coordinated and tested in both repositories.

Known limitations:

- Direct post generation requires an Anthropic key on each device.
- The iOS Simulator Keychain is per simulator.
- Generated posts only receive the stored summary, not full article text.
- Physical-device builds require manual signing configuration.
